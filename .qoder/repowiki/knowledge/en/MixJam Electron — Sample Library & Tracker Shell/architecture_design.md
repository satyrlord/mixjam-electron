- **Multi-target Build Orchestration**: `electron.vite.config.ts` defines distinct build pipelines for Main (including a dedicated `indexer` worker entry), Preload, and Renderer, ensuring correct asset bundling and dependency externalization across process boundaries.
- **Typed IPC Boundary Enforcement**: The `src/shared/ipc.ts` contract acts as the single source of truth for cross-process communication, enforced by the Preload's `contextBridge` exposure and consumed by both Main handlers and Renderer hooks (`useAppState`).
- **Native Module Lifecycle Management**: The root `package.json` scripts (`predev`, `prebuild`) mandate a synchronized `electron-rebuild` step for `better-sqlite3`, ensuring native bindings match the Electron runtime before any development or production build starts.
- **Shared Design Token Distribution**: The `public/themes/` directory serves as a static asset hub for JSON-based design tokens, consumed by the frontend style system to drive CSS custom properties without requiring build-time transformation.

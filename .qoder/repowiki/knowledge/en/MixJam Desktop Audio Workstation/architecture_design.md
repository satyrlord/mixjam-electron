- Multi-target build orchestration: `electron.vite.config.ts` declares three targets (main with an additional `indexer` worker entry, preload, renderer) so each process bundles only its own dependencies via `externalizeDepsPlugin`, while a custom `inject-csp` plugin injects a strict Content-Security-Policy meta tag at build time for the packaged renderer.
- Typed IPC boundary enforcement: The shared contract in `src/shared/ipc.ts` is the single source of truth consumed by Main handlers, Preload's `contextBridge` exposure, and Renderer hooks (`useAppState`), keeping cross-process calls type-safe across all three targets.
- Native module lifecycle gating: Root `package.json` scripts chain `predev`/`prebuild` → `rebuild:electron` → `electron-rebuild -f -w better-sqlite3`, and `pretest`/`pretest:*` → `rebuild:node` (direct `node-gyp rebuild` against local Node), ensuring the native SQLite binding matches whichever runtime is executing.
- Shared design token distribution: `public/themes/*.json` files are static assets served to the renderer at runtime; they drive the CSS custom property theme system without any build-time transformation, keeping skinning decoupled from bundling.
- Version pinning across boundaries: The root config reads `package.json` version at build time and defines it as `__APP_VERSION__`, making the same version string available to both the main process and the renderer footer/mixjam.json.
